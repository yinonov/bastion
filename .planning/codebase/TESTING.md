## Test Structure

**Suite Organization:**
```typescript
import test from "node:test";
import assert from "node:assert/strict";

test("blocks dangerous bash commands", () => {
  const result = evaluatePolicy(event, config);
  assert.equal(result.decision.decision, "deny");
  assert.ok(result.findings.some((finding) => finding.type === "dangerous_command"));
});
```
Pattern source: `packages/core/src/policy.test.ts`.

**Patterns:**
- Setup pattern: inline fixture objects inside each test, sometimes with generated UUIDs (`packages/core/src/policy.test.ts`, `packages/insights/src/insights.test.ts`).
- Teardown pattern: explicit cleanup for temp resources (`mkdtemp`, `rm`) in `packages/edge/src/server.test.ts`.
- Assertion pattern: primitive/assertion helpers (`assert.equal`, `assert.ok`, `assert.match`) across all package tests.

- Current suite favors lightweight integration tests over mocking.

**What NOT to Mock:**
- Core policy and redaction pure logic (`packages/core/src/policy.ts`, `packages/core/src/redaction.ts`) should remain real-function unit tests.
- SQLite store schema parsing/serialization in `packages/edge/src/store.ts` benefits from real DB behavior for correctness checks.

## Common Patterns

**Async Testing:**
```typescript
test("denies unapproved MCP upstreams", async () => {
  const app = await createEdgeApp(getDefaultConfig(), store);
  const response = await app.inject({ method: "POST", url: "/mcp/unapproved", payload: {...} });
  assert.equal(response.statusCode, 403);
  await app.close();
});
```
Pattern source: `packages/edge/src/server.test.ts`.

**Error Testing:**
```typescript
const result = evaluatePolicy(event, config);
assert.equal(result.event.status, "denied");
assert.ok(result.findings.some((finding) => finding.type === "dangerous_command"));
```
Pattern source: `packages/core/src/policy.test.ts`.

## Current Testing Gaps

- Dashboard package has no real tests; `apps/dashboard/package.json` uses a placeholder success command (`node -e "process.exit(0)"`).
- CLI test coverage is minimal and non-behavioral (`packages/cli/src/install-hooks.test.ts` only checks a string include).
- No tests found for several high-impact modules:
  - `packages/cli/src/index.ts`
  - `packages/edge/src/store.ts`
  - `packages/core/src/hooks.ts`
  - `apps/dashboard/lib/api.ts`
  - `apps/dashboard/app/page.tsx`
- No coverage reporting or thresholds means regressions can pass undetected.
- No explicit failure-path assertions using `assert.throws`/`assert.rejects` for parsing/network/file I/O boundaries.
- No mock strategy for outbound MCP HTTP proxy behavior in `packages/edge/src/server.ts`.
