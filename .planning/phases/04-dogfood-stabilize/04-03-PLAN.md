  <action>In the `LocalSqliteStore` class migration logic, add a new table `uptime_log`:

```sql
create table if not exists uptime_log (
  id text primary key,
  event_type text not null, -- 'startup' or 'shutdown'
  timestamp text not null,
  uptime_seconds integer, -- duration if shutdown; null if startup
  crash_detected boolean default 0,
  metadata_json text
);
create index if not exists idx_uptime_log_timestamp on uptime_log(timestamp desc);
```

Add a public method `logUptime(eventType: 'startup' | 'shutdown', uptimeSeconds?: number, crashDetected?: boolean): void` that:
- Generates a UUID for the log entry
- Records the current ISO timestamp
- Stores uptimeSeconds if available (for shutdown events)
- Stores crash_detected flag (will be false in normal operation, true if the process was killed without graceful shutdown)
- Inserts into uptime_log

Also add a query method `getUptimeStats(): { startupCount: number, totalUptimeSeconds: number, lastShutdown: string | null, currentSession: number }` that:
- Counts 'startup' events
- Sums all uptime_seconds from 'shutdown' events
- Returns the most recent shutdown timestamp
- Calculates current session uptime = (now - most recent startup timestamp)

Returns zero-valued object if table is empty (no uptime data yet).</action>

<task type="auto">
  <name>Task 2: Hook edge server startup and graceful shutdown to log uptime</name>
  <files>packages/edge/src/server.ts</files>
  <action>In the `startEdgeServer()` function (in server.ts), after the server is created and listening:

1. **On startup:** Call `store.logUptime('startup')` after the Fastify server is successfully listening. Log a console message with the startup timestamp so the founder sees when the server started.

2. **On graceful shutdown:** Register handlers for SIGTERM and SIGINT signals that:
   - Calculate uptime in seconds since server started (now - startup timestamp)
   - Call `store.logUptime('shutdown', uptimeSeconds)` to record graceful shutdown
   - Close the server cleanly
   - Exit process with code 0

Example pattern (integrate into existing signal handlers if they exist):
```typescript
const startTime = Date.now();
let sessionUptime = 0;

app.listen({ port, host }, () => {
  store.logUptime('startup');
  console.log(`Bastion edge listening at ${url}`);
});

process.on('SIGTERM', () => {
  sessionUptime = Math.floor((Date.now() - startTime) / 1000);
  store.logUptime('shutdown', sessionUptime);
  app.close();
});
```

Do NOT add uptime tracking to the hot path (hook handler); only record at startup/shutdown.</action>

```
program
  .command("status")
  .description("Check Bastion edge server uptime and health")
  .action(async () => {
    const config = await loadConfig();
    const sqlitePath = resolveSqlitePath(config);
    
    // Open store connection temporarily to query uptime_log
    const { LocalSqliteStore } = await import("@bastion/edge");
    const store = new LocalSqliteStore(sqlitePath);
    const stats = store.getUptimeStats();
    
    console.log(`\n=== Bastion Edge Status ===`);
    console.log(`Total startups: ${stats.startupCount}`);
    console.log(`Cumulative uptime: ${formatSeconds(stats.totalUptimeSeconds)}`);
    console.log(`Current session uptime: ${formatSeconds(stats.currentSession)}`);
    console.log(`Last shutdown: ${stats.lastShutdown ?? "Never"}`);
    
    if (stats.totalUptimeSeconds >= 7 * 24 * 3600) {
      console.log(`\n✓ DOG-03 criterion met: ≥7 days cumulative uptime achieved`);
    } else {
      const remaining = (7 * 24 * 3600) - stats.totalUptimeSeconds;
      console.log(`\nUptime until DOG-03 criterion: ${formatSeconds(remaining)}`);
    }
  });

// Helper function
function formatSeconds(seconds: number): string {
  const days = Math.floor(seconds / 86400);
  const hours = Math.floor((seconds % 86400) / 3600);
  const mins = Math.floor((seconds % 3600) / 60);
  return `${days}d ${hours}h ${mins}m`;
}
```

This command can be run anytime to check uptime progress during the dogfood window. Display progress toward the ≥7 day criterion.</action>
